# 4. Purpose of Each File (Summary)
4.1 api/
	client.js
	Central axios instance: sets baseURL, attaches auth headers (token, HMAC), and handles interceptors.

	systemAuth.js
	Any system-level/auth bootstrap logic separate from normal user login.

api/auth/

	login.js — User login API call.

	signup.js — New user signup/registration.

	validateUser.js — Check if a user/account exists.

	accountVerification.js — Additional account validation.

	validateOTP.js — Verify OTP sent to user.

	forgetPassword.js — Initiate forgot password flow.

	changePass.js — Change password for logged-in user.

	kycOTP.js — OTP related to KYC validation.

	index.js — Aggregator/re-export for auth APIs.

api/customer/

	getCustomerDetails.js — Fetch customer core profile details.

	getCustomerIncome.js — Fetch stored income details.

	updateCustomerIncome.js — Save/Update income details (after user input + camera upload).

	submitKycDetails.js — Post full KYC payload at end of wizard.

	loan.js — Loan operations (initiation, general loan APIs).

	loanDetails.js — Specific loan details / statements / schedules.

	index.js — Aggregator for customer/loan-related APIs.

api/master/

	masterDataService.js — Fetch master data (province, district, etc.).

	masterDataStore.js — Local caching / in-memory store for master data.

4.2 apicontext/

	AuthContext.js
	React Context managing:

	Logged-in user object

	Auth token

	login(), logout(), restoreSession()

	Exposed via useAuth hook

4.3 apihook/

	useAuth.js
	Custom hook that wraps AuthContext to simplify:

	Accessing user

	Calling login / logout / changePassword etc.

4.4 assets/

	app_icon.png — App icon image.

	fonts/Poppins-*.ttf — Full Poppins font family used globally.

	images/logo-caption.png — App logo used on login/splash/headers.

4.5 config/

	appConfig.ts

	Reads environment variables (e.g. API base URL)

	Exports configuration constants used by client.js and others

4.6 hooks/

	useHandleAuthError.js
	A utility hook to interpret auth-related API errors and – for example –:

	Show global alerts

	Redirect to login if token expired

4.7 navigation/

	AppNavigator.js

		Root createNativeStackNavigator

	Handles:

		Splash, Login, Signup, ForgotPassword, OtpVerification

		MainDrawer (DrawerNavigator)

		UserProfile, ChangePassword

		Loan screens (QuickLoan, NormalLoan, LoanStatus, LoanStatement, CustIncomeScreen as IncomeDetails, Affordability)

		KYC screens (KYCDetails and all KYC wizard steps)

		KYC screens have custom header title styles and a Home icon that warns and sends the user back to the main drawer.

	DrawerNavigator.js

		createDrawerNavigator after login

	Provides:

		Home screen with custom brand-like header

		UserProfile

		ChangePassword

		Logout drawer item

		Shows user’s name in the drawer header and in headerRight of Home.

4.8 screens/

	Main screens:

	SplashScreen.js — App start screen, checks session and routes to Login or MainDrawer.

	LoginScreen.js — Username/password login, calls auth APIs, saves token/user.

	SignupScreen.js — New user registration flow (mobile/account-based).

	ForgotPasswordScreen.js — Start reset password (OTP to mobile/email).

	OtpVerificationScreen.js — OTP validation screen for signup/reset flows.

	HomeScreen.js — Dashboard landing screen after login.

	QuickLoanScreen.js — Faster/simpler loan initiation flow.

	NormalLoanScreen.js — Full loan journey with more details.

	CustIncomeScreen.js  — Capture and edit customer income (with camera upload & formatted amounts).

	AffordabilityScreen.js — Current module; user enters Tenure + Loan Amount, calls API (later) and displays eligible products horizontally.

	LoanStatusScreen.js — View current loan application status.

	LoanStatementScreen.js — View list or schedule of repayments.

	UserProfileScreen.js — Display user’s profile details.

	ChangePasswordScreen.js — Change password for logged in user.

	KYCDetailsScreen.js — Starting point for KYC flow (summary/entry).

	KYC sub-screens (screens/kyc/):

		KycContext.js — (If used) keeps KYC data across steps.

		KycMobileScreen.js — Capture/verify mobile number.

		KycEmailScreen.js — Capture/verify email address.

		KycAddressScreen.js — Address details incl. Province, District, etc.

		KycContactScreen.js — Additional contact details.

		KycDocumentsScreen.js — Upload KYC documents (using camera/crop picker).

		KycReviewScreen.js — Final summary; user reviews and confirms before submitKycDetails API.

4.12 utils/

	auth.js — HMAC signing helpers, building secure auth headers.

	secureStore.js — Wrapper for react-native-encrypted-storage (save/get/remove sensitive values).

	session.js — Helpers to restore session at app start, check token presence, etc.

	dateUtils.js — Date formatting and parsing, typically via date-fns.

	interpretAccountStatus.js — Converts backend “status codes” into human-readable status messages.

# 5. App Overview

MLIS (Mobile Loan & Insurance System) is a React Native mobile application that enables customers to manage their loan journey from a smartphone.

Key capabilities:

User authentication:

	Login with username/password

	Signup with OTP verification

	Forgot password + OTP

Customer profile & KYC:

	Multi-step KYC wizard (mobile, email, address, documents, review)

	Profile view and update

Income capture & verification:

	Capture payslip via camera (image picker + crop)

	Enter income details with amount formatting

	Update income details through APIs

Loan operations:

	Quick Loan and Normal Loan initiation flows

Affordability check:

	Inputs: Tenure (months), Loan Amount

	Output: Eligible products with ROI, max amount, affordable flag

Loan status view

Loan statement / schedule view

Security & configuration:

Backend integrated using HMAC-secured APIs

Secure token storage via encrypted storage

10. Notes / Special Decisions

Project uses React Native 0.82.1 with Hermes enabled.

Using react-native-reanimated 4.1.5 and react-native-worklets 0.6.1 — these are sensitive to Gradle/RN versions and may cause build issues if upgraded blindly.

Custom font family Poppins is fully integrated under assets/fonts.

Affordability Screen currently uses mock data for products until backend API is ready, but the UI & state structure is in place.

HMAC keys and base URL are configured via .env and must be kept out of version control.
